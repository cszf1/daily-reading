## 文章一：数组中重复元素的 O(N) 查找算法

> 原文：[Finding a duplicated item in an array of N integers in the range 1 to N − 1](https://devblogs.microsoft.com/oldnewthing/20260413-00/?p=112227)
> 来源：Microsoft DevBlogs - The Old New Thing | 分类：Systems

### 核心内容

一位同事告诉我，有一种 **O(N) 算法**可以在包含 N 个整数的数组中找出重复元素（这些整数的范围是 1 到 N-1）。根据鸽巢原理，必然存在至少一个重复值。

**问题背景**：同事本以为这个问题只能通过排序达到 O(N log N) 的时间复杂度，作为面试题考察候选人。然而面试者找到了一个更优的**线性时间算法**！

**方法一（我的方案）**：将数组视为基于 1 的索引链表，利用每个整数的符号位作为访问标记。从索引 1 开始沿着索引跳转，直到找到符号位已被设置的元素（即重复值），或者回到索引 1 形成循环。这种方法会修改数组值，但最后会恢复原始数据。

**方法二（面试者更优的方案）**：完全不修改数组。核心思路是利用 Floyd 环检测算法（Floyd's cycle detector algorithm，即"龟兔赛跑"算法）。关键洞察是：数组中没有元素值为 N，所以索引 N 不可能是环的一部分。因此，从 N 开始的链必然会连接到某个已存在的环，而那个连接点就是重复元素所在位置。

**进阶技巧**：如果限制恰好只有一个重复值，可以直接通过对所有值求和，再减去 N(N-1)/2 来得到答案。

### 技术要点

- **时间复杂度**：O(N) 线性时间
- **空间复杂度**：O(1)（方法二，不修改原数组）
- **核心算法**：Floyd 环检测算法（龟兔赛跑算法）
- **关键洞察**：利用数组值作为索引，将问题转化为链表环检测

### 代码示例

```python
def find_duplicate(nums):
    """Floyd's Tortoise and Hare Algorithm"""
    # 阶段一：找到环的交点
    tortoise = nums[0]
    hare = nums[0]
    while True:
        tortoise = nums[tortoise]
        hare = nums[nums[hare]]
        if tortoise == hare:
            break
    
    # 阶段二：找到环的起点（重复元素）
    tortoise = nums[0]
    while tortoise != hare:
        tortoise = nums[tortoise]
        hare = nums[hare]
    
    return tortoise
```

### 对电赛的价值

- **图像处理中的数据去重**：处理传感器数据或图像特征点时，常需要对重复检测进行优化
- **嵌入式资源受限场景**：O(1) 空间复杂度对于 MCU/STM32 等内存受限设备尤为重要
- **算法思维训练**：将数组索引转化为链表环的思想可应用于状态机分析、依赖图检测等场景

## 文章二：DRAM 价格正在扼杀爱好者单板计算机市场

> 原文：[DRAM pricing is killing the hobbyist SBC market](https://www.jeffgeerling.com/blog/2026/dram-pricing-is-killing-the-hobbyist-sbc-market/)
> 来源：Jeff Geerling Blog | 分类：Hardware

### 核心内容

树莓派官方近日宣布，**所有配备 LPDDR4 内存的 Pi 型号再次涨价**，同时推出了"恰到好处"的 3GB 树莓派 4，售价 $83.75。涨价后，16GB 树莓派 5 的价格已达到 **$299.99**。这不是愚人节玩笑。

**市场现状分析**：

1. **LPDDR 芯片占成本大头**：从各供应商反馈来看，LPDDR 芯片现在占主板成本的大部分
2. **新产品线几乎停滞**：除了 Radxa 等少数厂商仍有新品发布节奏，大多数 SBC 厂商已停止推出新板子
3. **迷你 PC 也不再便宜**：曾经性价比很高的迷你 PC，现在 8GB 型号也要 $250+
4. **二手市场水涨船高**：超过 4GB 内存的二手 PC 价格也大幅上涨

**对爱好者社区的影响**：

- 作者大多数项目的设计目标是成本低于 $100
- 便宜的学习板子能让学习者不会因为弄坏零件而过度担心
- 随着价格上涨，能做的项目类型受到限制
- 作者表示现在更多转向使用老款 SBC 和微控制器

**展望**：树莓派公司的 Eben Upton 表示"内存价格不会永远维持在目前的高水平，挑战只是暂时的"。但问题是：我们需要等待多久？在那之前，爱好者 SBC 市场还能存在吗？

幸运的是，树莓派还有微控制器生态和工业基础支撑。作者担心更小的厂商可能无法长期坚持。

### 技术要点

- **LPDDR4/5 内存价格持续上涨**
- **SBC（单板计算机）成本结构变化**
- **微型化与成本控制的平衡**
- **替代方案：老款 SBC + 微控制器**

### 对电赛的价值

- **电赛硬件选型参考**：在预算有限的情况下，可以考虑：
  - ESP32 系列（内置 WiFi/BT，价格低廉）
  - STM32F4 系列（性能强劲，外设丰富）
  - 国产替代方案（如 GD32、CH32）
- **成本意识**：电赛通常有预算限制，了解当前硬件市场有助于做出最优选型
- **备用方案准备**：如树莓派价格过高，可以考虑 Orange Pi、Rock Pi 等替代品

## 文章三：watgo - Go 语言的 WebAssembly 工具包

> 原文：[watgo - a WebAssembly Toolkit for Go](https://eli.thegreenplace.net/2026/watgo-a-webassembly-toolkit-for-go/)
> 来源：Eli Bendersky's Blog | 分类：Systems

### 核心内容

作者发布了 **watgo**（WebAssembly Toolkit for Go），这是一个用纯 Go 编写的零依赖 WebAssembly 工具包，类似于 C++ 的 wabt 或 Rust 的 wasm-tools。

**核心功能**：

| 功能 | 描述 |
|------|------|
| Parse | 将 WAT（WebAssembly Text）解析为 wasmir |
| Validate | 使用官方 WebAssembly 验证语义检查模块 |
| Encode | 将 wasmir 编码为 WASM 二进制格式 |
| Decode | 从二进制格式读取 WASM 到 wasmir |

**wasmir** 是 WebAssembly 模块的语义表示，用户可以检查和操作它。

**CLI 使用示例**：

```bash
go install github.com/eliben/watgo/cmd/watgo@latest
watgo parse stack.wat -o stack.wasm
```

**Go API 示例**：

```go
package main

import (
    "fmt"
    "github.com/eliben/watgo"
    "github.com/eliben/watgo/wasmir"
)

func main() {
    m, err := watgo.ParseWAT([]byte(wasmText))
    if err != nil {
        panic(err)
    }
    
    // 分析模块中的函数、参数、指令
    for _, fn := range m.Funcs {
        sig := m.Types[fn.TypeIdx]
        for _, param := range sig.Params {
            if param.Kind == wasmir.ValueKindI32 {
                i32Params++
            }
        }
        // ... 遍历函数体指令
    }
}
```

**测试策略**：watgo 使用 WebAssembly 官方测试套件进行端到端测试，包含近 200K 行 WAT 文件，覆盖各种执行语义和错误场景。watgo 通过了全部 WASM 规范核心测试用例。

### 技术要点

- **零依赖 Go 实现**：无外部库依赖，便于集成
- **WAT ↔ WASM 双向转换**
- **完整的语义验证**
- **官方测试套件覆盖**
- **支持最新 WASM 提案**（包括 Garbage Collection）

### 对电赛的价值

- **边缘计算部署**：WASM 可以在边缘设备上运行编译后的模块
- **跨平台图像处理**：将图像处理算法编译为 WASM，可在多种嵌入式平台运行
- **性能关键代码**：WASM 提供接近原生的执行效率
- **工具链开发**：学习 WASM 工具链原理，有助于理解编译优化过程
