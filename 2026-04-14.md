## 文章一：用树莓派搭建自己的拨号ISP

> 原文：[Build your own Dial-up ISP with a Raspberry Pi](https://www.jeffgeerling.com/blog/2026/build-your-own-dial-up-isp-with-a-raspberry-pi/)
> 来源：jeffgeerling.com | 分类：Hardware

### 核心内容

这是一个非常有趣的硬件项目，作者用树莓派搭建了一个复古的拨号ISP，并将其与WiFi结合。

**硬件配置：**
- SBC（40美元）：树莓派 3、4 或 5
- 电话线路模拟器（120美元）：Viking DLE-200B双向线路模拟器
- 调制解调器（45美元）：StarTech.com 56K USB拨号调制解调器

**软件架构：**
项目使用两个Linux工具协同工作：

1. **mgetty**（modem get tty）：处理调制解调器的来电协商，连接建立后将控制权交给PPP

2. **PPP**（点对点协议）：认证远程计算机，然后在两台计算机之间配置网络桥接

**核心配置文件示例：**
```
Mar 26 15:32:35 dialpi pppd[15926]: PAM Account OK for dial
Mar 26 15:32:35 dialpi pppd[15926]: user dial logged in on tty ttyACM0 intf ppp0
Mar 26 15:32:35 dialpi pppd[15926]: PAP peer authentication succeeded for dial
Mar 26 15:32:35 dialpi pppd[15926]: Peer dial authenticated with PAP
```

**技术亮点：**
- 使用AT命令（如`ATA`接听来电、`AT+MS`设置最大速率）直接与调制解调器交互
- 可通过`minicom -D /dev/ttyACM0`手动控制调制解调器
- 最多可获得33.6Kbps的稳定连接（28.8Kbps更稳定）
- 使用Macproxy Classic代理服务器让老式电脑也能浏览现代网站

### 技术要点

- **调制解调器通信**：串口通信、AT命令集、PAP认证协议
- **PPP协议栈**：点对点隧道协议、网络桥接配置
- **QAM调制**：解释了 quadrature amplitude modulation 如何应用于WiFi
- **Linux系统工具**：mgetty守护进程、pppd配置、journalctl日志查看

### 对电赛的价值

### 核心内容

作者宣布发布 watgo —— 一个纯Go语言编写的WebAssembly工具包，类似于wabt（C++）或wasm-tools（Rust），但零依赖。

**核心功能：**
- **Parse**：将WAT（WebAssembly Text）解析为中间表示
- **Validate**：使用官方WebAssembly验证语义检查模块是否格式正确
- **Encode**：将中间表示编码为WASM二进制格式
- **Decode**：从二进制格式读取并解码

**API使用示例：**
```go
package main

import (
  "fmt"
  "github.com/eliben/watgo"
  "github.com/eliben/watgo/wasmir"
)

const wasmText = `
(module
 (func (export "add") (param i32 i32) (result i32)
 local.get 0
 local.get 1
 i32.add
 )
)`

func main() {
  m, err := watgo.ParseWAT([]byte(wasmText))
  if err != nil {
    panic(err)
  }
  
  // 分析模块中的函数
  for _, fn := range m.Funcs {
    sig := m.Types[fn.TypeIdx]
    for _, instr := range fn.Body {
      switch instr.Kind {
      case wasmir.InstrLocalGet:
        localGets++
      case wasmir.InstrI32Add:
        i32Adds++
      }
    }
  }
}
```

- 同时使用wabt的interp测试套件验证

- **编译器前端技术**：词法分析、语法解析、语义验证

## 文章三：Value Numbering（值编号）

> 原文：[Value numbering](https://bernsteinbear.com/blog/value-numbering/)
> 来源：bernsteinbear.com | 分类：Programming | 点赞：23

### 核心内容

这是一篇关于编译器优化的深度技术文章，讲解了如何识别并消除重复计算。

**核心概念：**

1. **SSA（静态单赋值形式）**：为每个表达式分配唯一的变量名
```
# 原始代码
x = 0
x = x + 1
x = x + 1

# SSA形式
v0 = 0
v1 = v0 + 1
v2 = v1 + 1
```

2. **值编号（Value Numbering）**：识别在编译时已知必定产生相同值的指令

**局部值编号（Local Value Numbering）：**
```python
# 伪代码实现
value_map = ValueMap()
for instr in block.instructions:
    if instr.wants_value_numbering():
        if instr.value_number() in value_map:
            # 替换为已有的值
            replace_with(value_map[instr.value_number()])
        else:
            # 添加到映射表
            value_map[instr.value_number()] = instr
```

**哈希优化（Hash-Consing）：**
```java
public int valueNumber() {
    return 0x20000000
    | (opcode
       + 7  * System.identityHashCode(x)
       + 11 * System.identityHashCode(y));
}

public boolean valueEqual(Instruction i) {
    if (i instanceof Op2) {
        Op2 o = (Op2) i;
        return opcode == o.opcode && x == o.x && y == o.y;
    }
    return false;
}
```

**全局值编号（Global Value Numbering）：**
- 需要处理控制流（分支、循环等）
- 使用支配者（Dominator）关系来决定值映射的传播
- 在反向后序遍历中迭代基本块

**纯函数 vs 非纯函数：**
- 纯函数：不与外部世界交互（如算术运算）
- 非纯函数：I/O操作、内存加载等（不能直接应用值编号）
